Engineering
Philosophy

Architecture before code.

Every system begins with a clear threat model — failure modes, data consistency boundaries, and operational bottlenecks mapped before a single line is written. Complexity is managed through constraint, not cleverness.

Operational readiness is a feature.

Software that works on localhost and software that survives production are two different products. Deployment pipeline, environment configuration, and failure recovery are built into the spec — not bolted on after.

The system is the product.

Individual components are not deliverables. A working, monitored, documented system that a non-technical operator can maintain without calling the developer — that is the deliverable.

Cloud Infrastructure Video Processing Mobile
Achieved sub-2-second live feed latency with a resilient offline queue guaranteeing zero archive frame loss across unstable network conditions.

VSaaS — Video Surveillance as a Service

The Constraint

RTSP camera networks installed in retail environments operate over consumer-grade Wi-Fi — connections that drop, degrade, and recover unpredictably throughout the day. A naive streaming pipeline drops footage whenever connectivity lapses. The client requirement was unambiguous: zero frame loss in the historical archive, with live feeds that immediately snap back to real-time on reconnection.

Engineering Architecture

A Python bridge process ingests raw RTSP feeds and operates two simultaneous FFmpeg encoding pipelines: a rolling 2-second HLS playlist for live consumption, and 60-second .ts segment files for archival chunking. An intelligent thread-safe queue intercepts the upload path. During network outages, segments cache locally and are flushed to MinIO object storage in strict temporal order once connectivity resumes — preserving timeline integrity without operator intervention.

Live feeds intentionally discard stale frames during outages rather than serve delayed footage, ensuring the live view represents reality the moment it returns. The mobile application, built with React Native and Expo, stitches daily chunk sequences into continuous playback timelines dynamically queried from Supabase PostgreSQL metadata.

Stack

Python · FFmpeg · HLS/RTSP · MinIO S3 · Supabase (PostgreSQL) · React Native · Expo · expo-av · Docker Compose

Enterprise SaaS POS System Full-Stack
Replaced fragmented paper-based and spreadsheet workflows with a unified POS + repair management system, consolidating four operational roles under role-based access control.

RSManager — Repair Shop & Point of Sale Platform

The Constraint

Electronics repair shops operate across at least four distinct operational surfaces simultaneously: customer checkout (POS), device repair tracking, inventory management, and supplier procurement. Each surface had its own informal system — receipts, notebooks, spreadsheets — producing reconciliation errors, inventory blindspots, and no unified customer history. A single platform needed to serve a non-technical owner, a technician on the floor, and a cashier at the counter without training overhead.

Engineering Architecture

The backend is a FastAPI service with asynchronous SQLAlchemy ORM on PostgreSQL, secured with JWT authentication and bcrypt-hashed credentials. Four role tiers — Owner, Manager, Technician, Cashier — enforce API-level permission boundaries, not UI-level hiding. The POS module supports barcode scanning, trade-in discounts, custom line items, and thermal receipt output via python-escpos. Repair jobs move through a configurable Kanban pipeline (Received → Diagnosed → Awaiting Parts → Ready → Collected) with timestamped audit logs. Inventory management enforces configurable low-stock thresholds and auto-generates supplier purchase orders.

The frontend is a React + Vite application using TypeScript, TailwindCSS, and Zustand for global state. The full application bundles into a portable offline distribution for shops without reliable internet.

Stack

FastAPI · Python · SQLAlchemy (Async) · PostgreSQL · JWT · bcrypt · ReportLab · python-escpos · React · TypeScript · Vite · TailwindCSS · Zustand

IoT Systems Mobile Engineering Real-Time Data
Designed end-to-end IoT-to-mobile telemetry pipeline delivering real-time pediatric health vitals to parents with configurable threshold alerting and AI-assisted interpretation.

HCMonitor — IoT Wearable Health Monitoring System

The Constraint

A custom hardware wearable device collects continuous biometric data from children — body temperature and GPS coordinates — and must surface that data to a parent's phone with no perceptible lag. The challenge is bridging the hardware-to-cloud-to-mobile data path reliably while keeping the parent-facing interface simple enough to require no documentation. Abnormal readings must trigger immediate, context-aware alerts before the parent checks the screen.

Engineering Architecture

Sensor data from the wearable transmits wirelessly to a cloud ingestion layer. The React Native mobile application — built with Expo SDK 54, React Navigation v7, and TypeScript — provides a live dashboard rendering temperature history as time-series charts (react-native-chart-kit) and real-time GPS position on an interactive map (react-native-maps). An AI chatbot interface interprets raw readings in plain language, allowing parents to ask health-context questions without needing medical literacy. The application supports multiple registered devices under one parent account, with a PIN-secured authentication context.

Stack

React Native · Expo SDK 54 · TypeScript · React Navigation v7 · expo-location · react-native-maps · react-native-chart-kit · EAS Build · IoT sensor integration · AI API

Data Engineering Automation Lead Generation
Automated extraction of Google Maps leads for dz_schools platform, bypassing dynamic anti-bot measures to build a comprehensive database of Algerian private schools.

DZ Lead Gen Pro — Automated Maps Scraper

The Constraint

The client needed a comprehensive database of private schools across all 58 Algerian wilayas, specifically targeting institutions without existing websites. Manual data entry was unfeasible, and standard scraping scripts were immediately blocked by Google Maps' dynamic loading algorithms, infinite scroll behaviors, and stringent anti-bot measures.

Engineering Architecture

An asynchronous Python application leverages the Playwright library to launch a headed Chromium browser with automation-controlled flags disabled, effectively mimicking human traversal patterns. The scraper navigates region-by-region, programmatically handling consent screens and triggering deliberate scroll events to force dynamic payload rendering. A Streamlit frontend provides a clean, responsive UI for non-technical users to configure parameters such as target wilayas, execution delays, and exclusion filters, subsequently generating a clean CSV output.

Stack

Python · Playwright (Async) · Streamlit · Pandas · asyncio

Healthcare Ops Algorithm Design Institutional Tool
Eliminated scheduling bias and holiday-assignment disputes across medical on-call rosters by implementing a stateful fairness-guaranteed round-robin algorithm with historical memory.

DocsGuards — Medical On-Call Scheduling Engine

The Constraint

Hospital on-call scheduling is contentious territory: doctors track who was last assigned a major holiday, who has been on leave, and whether the standard-night pool is being treated fairly versus the weekend pool. Manual spreadsheet scheduling introduces implicit bias, and any mistake in fairness tracking creates institutional conflict. The system needed mathematically provable fairness, not just perceived fairness.

Engineering Architecture

A custom continuous round-robin queuing engine maintains separate state machines for Standard (weekday night) and Weekend pools. The algorithm enforces a hard rule: no doctor receives a second assignment until every eligible, non-leave doctor has received one — across both single-night and multi-day scheduling horizons. A Holiday Memory Engine stores per-holiday assignment history, ensuring the same named holiday (Eid Al-Adha, National Day) rotates equitably across the full pool over multi-year periods. Leave window checks transparently skip unavailable doctors without corrupting their queue position. Schedules export to dynamically generated, colour-coded Excel files via openpyxl for institutional distribution.

Stack

Python 3.8+ · Flask · Custom Round-Robin State Engine · openpyxl · SQLite · HTML/CSS (Inter)
